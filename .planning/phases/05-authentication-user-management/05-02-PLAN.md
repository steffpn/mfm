1. Create test helper `apps/api/tests/helpers/auth.ts`:
   - Export `async function createTestAdmin(fastify)`: Creates an admin user with hashed password directly in DB using prisma. Returns { user }.
   - Export `async function getAuthTokens(fastify, email, password)`: Calls POST /api/v1/auth/login via server.inject to get tokens. Returns { accessToken, refreshToken, user }.
   - Export `async function createTestUserWithTokens(fastify, role, email)`: Creates user directly in DB, generates tokens, returns { user, accessToken, refreshToken }.
   - This helper will be imported by auth tests, admin tests, and endpoint protection tests in Plan 03.

2. Create `apps/api/src/routes/v1/auth/schema.ts`:
   - `RegisterSchema`: body with code (string, minLength 14, maxLength 14), email (string, format email), password (string, minLength 8), name (string, minLength 1, maxLength 100). Use TypeBox Type.Object.
   - `LoginSchema`: body with email (string, format email), password (string, minLength 1).
   - `RefreshSchema`: body with refreshToken (string, minLength 1).
   - `LogoutSchema`: body with refreshToken (string, minLength 1).

3. Create `apps/api/src/routes/v1/auth/handlers.ts`:
   - `register`: Validate invitation code (findUnique by code, check status=PENDING, not expired, usedCount < maxUses). Check email uniqueness. Hash password with hashPassword(). Use prisma.$transaction to: create user with role from invitation, create UserScope if invitation.scopeId is set (entityType = invitation.role, entityId = invitation.scopeId), increment usedCount and set status to REDEEMED if usedCount+1 >= maxUses. Generate token pair. Return 201 with user (id, email, name, role) + tokens.
   - `login`: Find user by email. If not found or !isActive, return 401 "Invalid credentials". Verify password with verifyPassword(). If wrong, return 401. Update lastLoginAt. Generate token pair. Return 200 with user + tokens.
   - `refresh`: Find refresh token in DB by token value. Check not revoked (revokedAt is null), not expired, user is active. Revoke old token (set revokedAt=now). Generate new token pair. Return 200 with new tokens. If invalid, return 401 "Invalid refresh token".
   - `logout`: Extract current user from request (requires authenticate preHandler). Find refresh token by token value AND userId matching current user. Revoke it (set revokedAt=now). Return 200 { message: "Logged out" }.

4. Create `apps/api/src/routes/v1/auth/index.ts`:
   - Fastify plugin with: POST /register (schema: RegisterSchema), POST /login (schema: LoginSchema), POST /refresh (schema: RefreshSchema), POST /logout (preHandler: [authenticate], schema: LogoutSchema).
   - NOTE: register, login, refresh are PUBLIC (no auth). logout requires auth.

5. Update `apps/api/src/routes/v1/index.ts`:
   - Add: `fastify.register(import("./auth/index.js"), { prefix: "/auth" })`

   - Test refresh: valid refresh returns new tokens + rotates (old revoked), revoked refresh returns 401, expired refresh returns 401.
   - Test logout: with auth header + refresh token, revokes that token, returns 200.
