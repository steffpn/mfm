    3. **Digest worker** (`apps/api/src/workers/digest.ts`):
       - Follow cleanup.ts BullMQ pattern exactly: export `startDigestWorker()` returning `{ queue, worker }`
       - Queue name: "digest-notifications"
       - Two upsertJobScheduler calls: daily-digest-scheduler (cron "0 9 * * *", tz "Europe/Bucharest") and weekly-digest-scheduler (cron "0 9 * * 1", tz "Europe/Bucharest")
       - Worker dispatches by job.name: "daily-digest" -> processDailyDigests(), "weekly-digest" -> processWeeklyDigests()
       - `computeDailyDigest(stationIds: number[])`: Query daily_station_plays for yesterday (CURRENT_DATE - 1 day), sum play_count. Query airplay_events for yesterday's top song (GROUP BY song_title, artist_name ORDER BY count DESC LIMIT 1). Query daily_station_plays joined with stations for top station. Return typed result.
       - `computeWeeklyDigest(stationIds: number[])`: Query daily_station_plays for last 7 days and previous 7 days. Calculate week-over-week % change. Top song from airplay_events for last 7 days. Top station from daily_station_plays. Count new stations (stations that played this week but not the week before). Return typed result.
       - `processDailyDigests()`: Find all users where isActive=true AND dailyDigestEnabled=true, include scopes and deviceTokens. For each user, derive stationIds from scopes (STATION role: scoped IDs, others: all station IDs). Call computeDailyDigest. Build notification. Send via APNS to each device token. On BadDeviceToken/Unregistered error, delete token from DB. Wrap each user in try/catch -- one failure must not block others.
       - `processWeeklyDigests()`: Same pattern but weeklyDigestEnabled and computeWeeklyDigest.
       - Notification payload: { alert: { title, body }, data: { type: "daily_digest"|"weekly_digest", date: "YYYY-MM-DD" } }. Body follows locked format from CONTEXT.md.
       - Export computeDailyDigest and computeWeeklyDigest for direct testing.

       - Start digest worker after cleanup worker but before detection worker (digest is independent, order doesn't matter much but keep consistent)
       - Add to shutdown sequence: digestWorker.close() + digestQueue.close() after detection worker shutdown
