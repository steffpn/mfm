    3. **Handlers** (`apps/api/src/routes/v1/competitors/handlers.ts`):
       - `getWatchedStations`: Query WatchedStation where userId = currentUser.id, include station { name }. Map to response format.
       - `addWatchedStation`: Check count <= 20. Check stationId is not in user's own scopes. Prisma create with unique constraint catch (P2002 -> 409). Include station for response.
       - `removeWatchedStation`: Delete where userId AND stationId. If not found, 404.
       - `getCompetitorSummary`: Get all watched station IDs. For each, query daily_station_plays for period to get playCount. Query airplay_events for top song per station. Use a single SQL query with GROUP BY station_id for efficiency. Return array of cards.
       - `getCompetitorDetail`: Verify stationId is in user's watched list (403 if not). Three queries:
         a) Top songs: `SELECT song_title, artist_name, isrc, COUNT(*)::int AS play_count FROM airplay_events WHERE station_id = $1 AND started_at >= NOW() - interval GROUP BY ... ORDER BY play_count DESC LIMIT 20`
         b) Recent detections: `SELECT id, song_title, artist_name, started_at FROM airplay_events WHERE station_id = $1 ORDER BY started_at DESC LIMIT 50`
         c) Comparison: Get user's own station IDs from scopes. Find songs that appear on both competitor and own stations. `SELECT song_title, artist_name, SUM(CASE WHEN station_id = $competitor THEN 1 ELSE 0 END)::int AS their_plays, SUM(CASE WHEN station_id IN ($own_ids) THEN 1 ELSE 0 END)::int AS your_plays FROM airplay_events WHERE (station_id = $competitor OR station_id IN ($own_ids)) AND started_at >= NOW() - interval GROUP BY song_title, artist_name HAVING SUM(CASE WHEN station_id = $competitor THEN 1 ELSE 0 END) > 0 AND SUM(CASE WHEN station_id IN ($own_ids) THEN 1 ELSE 0 END) > 0 ORDER BY their_plays DESC LIMIT 20`
