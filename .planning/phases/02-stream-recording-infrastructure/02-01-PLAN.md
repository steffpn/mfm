<objective>
Station CRUD REST API with Redis pub/sub event publishing

Purpose: Enable admin to manage stations (add, edit, remove, list, bulk import) via REST API, with every mutation publishing a Redis pub/sub event that the supervisor service (Plan 02) will consume to start/stop/restart FFmpeg processes. This is the "control plane" for stream recording.

Output: Fastify route plugin at /api/v1/stations with TypeBox validation, Redis pub/sub helpers, and integration tests.
</objective>

The StationEvent payload shape: `{ stationId: number; streamUrl?: string; timestamp: string }`. The publish helper uses the existing `redis` import from `./redis.js` and calls `redis.publish(channel, JSON.stringify(payload))`.

    **2. Create `apps/api/src/routes/v1/stations/schema.ts`:**
    Define TypeBox schemas for request validation:
    - `StationCreateSchema`: Object with `name` (string, minLength 1), `streamUrl` (string, format 'uri'), `stationType` (Union of Literal 'radio' | 'tv'), `country` (Optional string, default 'RO')
    - `StationBulkCreateSchema`: Array of StationCreateSchema, minItems 1
    - `StationUpdateSchema`: Object with all fields Optional: `name`, `streamUrl`, `stationType`
    - `StationParamsSchema`: Object with `id` (Type.Number)
    - Response schemas for station objects

    **3. Create `apps/api/src/routes/v1/stations/handlers.ts`:**
    Route handler functions using Prisma for DB ops and pubsub for events:
    - `createStation`: Create with `status: 'ACTIVE'`, publish `station:added`, return 201
    - `createStationsBulk`: Use `prisma.$transaction` to create all stations, publish `station:added` for each, return 201
    - `listStations`: `prisma.station.findMany` selecting id, name, streamUrl, stationType, status, lastHeartbeat, restartCount
    - `getStation`: `prisma.station.findUnique`, return 404 if not found
    - `updateStation`: `prisma.station.update`, publish `station:updated` with new streamUrl. If station not found, return 404. Use try/catch on Prisma NotFoundError.
    - `deleteStation`: Set `status: 'INACTIVE'`, publish `station:removed`. If not found, return 404. Per user decision: soft delete only, DB record preserved.
