  <behavior>
    - Watchdog polls every 10 seconds (configurable)
    - For each stream with status 'recording', watchdog checks latest segment file mtime in station directory
    - If no segment file modified within 30 seconds (STALE_THRESHOLD_MS), watchdog triggers restartStream
    - If segment file is fresh and restartCount > 0, watchdog resets restartCount (stream recovered)
    - Healthy streams get their lastHeartbeat updated in DB
    - Watchdog also checks segment file size -- files under 1KB are treated as corrupt (not counted as fresh)
    - Watchdog.start() begins the polling interval, Watchdog.stop() clears it
    - Staggered startup loads all ACTIVE stations from DB and starts in batches of 10 with 2s delay between batches
    - Supervisor entry point subscribes to Redis pub/sub channels and dispatches to StreamManager
    - Supervisor graceful shutdown kills all FFmpeg processes, clears watchdog, disconnects Redis
  </behavior>
  <action>
    **1. Create `apps/api/src/services/supervisor/watchdog.ts`:**

    Export `Watchdog` class:
    - Constructor takes `StreamManager` and optional config: `{ intervalMs: 10_000, staleThresholdMs: 30_000, minSegmentBytes: 1024 }`
    - `start()`: Set up `setInterval` that calls `checkAll()` every `intervalMs`
    - `stop()`: Clear the interval
    - `async checkAll()`: Iterate over `streamManager.getAllStatuses()`. For each stream with status 'recording':
      1. Get segment directory path: `path.join(DATA_DIR, String(stationId))`
      2. List files in directory with `fs.readdir`
      3. For each file, `fs.stat` to get mtime and size
      4. Find the file with the latest mtime that is also >= minSegmentBytes in size
      5. If no qualifying file or latest mtime is older than staleThresholdMs: call `streamManager.restartStream(stationId)` -- stream is hung
      6. If qualifying file exists and is fresh: update `lastHeartbeat` in DB via Prisma, and if restartCount > 0 call `streamManager.resetRestartCount(stationId)`

    Export helper `getLatestSegmentMtime(segmentDir: string, minBytes: number): Promise<Date | null>` for reuse and testing.

    c. **Staggered startup** (per user decision: batch size 10, delay 2s):
    ```typescript
    const STARTUP_BATCH_SIZE = 10;
    const STARTUP_BATCH_DELAY_MS = 2_000;

    const stations = await prisma.station.findMany({ where: { status: 'ACTIVE' } });
    for (let i = 0; i < stations.length; i += STARTUP_BATCH_SIZE) {
      const batch = stations.slice(i, i + STARTUP_BATCH_SIZE);
      await Promise.all(batch.map(s => streamManager.startStream(s.id, s.streamUrl)));
      if (i + STARTUP_BATCH_SIZE < stations.length) {
        await new Promise(resolve => setTimeout(resolve, STARTUP_BATCH_DELAY_MS));
      }
    }
    ```

    d. **Start watchdog:** `watchdog.start()`

    e. **Redis pub/sub subscriber** (MUST use dedicated connection per ioredis requirement):
    ```typescript
    const subscriber = createRedisConnection();
    await subscriber.subscribe('station:added', 'station:removed', 'station:updated');
    subscriber.on('message', async (channel, message) => {
      const event: StationEvent = JSON.parse(message);
      switch (channel) {
        case 'station:added':
          await streamManager.startStream(event.stationId, event.streamUrl!);
          break;
        case 'station:removed':
          await streamManager.stopStream(event.stationId);
          break;
        case 'station:updated':
          // Stop-and-restart per user decision (brief gap acceptable)
          await streamManager.stopStream(event.stationId);
          await streamManager.startStream(event.stationId, event.streamUrl!);
          break;
      }
    });
    ```

    f. **Reconciliation on subscriber reconnect:** When Redis subscriber reconnects (ioredis emits 'connect' after reconnect), do a full DB reconciliation: load all ACTIVE stations, compare with running streams, start missing ones, stop extras. This handles pub/sub message loss per Pitfall 4.

    g. **Graceful shutdown:**
    Register SIGTERM/SIGINT handlers:

    ```typescript
      await streamManager.stopAll();
      subscriber.disconnect();
      await prisma.$disconnect();
      process.exit(0);
    };
    process.on('SIGTERM', shutdown);
    process.on('SIGINT', shutdown);
    ```
