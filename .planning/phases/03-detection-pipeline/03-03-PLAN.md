    - Two callbacks with null ISRC but same normalized title+artist + same station + 2 minutes apart -> one AirplayEvent with playCount=2
    - Two callbacks with null ISRC and different normalized title+artist -> two separate AirplayEvents

    - Callback with lower confidence does NOT update AirplayEvent metadata (keeps existing)

    - Callback with data.status.code=1001 creates NoMatchCallback record (stationId, callbackAt, statusCode)
    - Callback with data.status.code=1001 does NOT create a Detection or AirplayEvent
    - Callback with empty music array (no metadata.music or music=[]) creates NoMatchCallback

    Idempotency:
    - Duplicate rawCallbackId (same stream_id+timestamp_utc) is handled gracefully (unique constraint catch)
