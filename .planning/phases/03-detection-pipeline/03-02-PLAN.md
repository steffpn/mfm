    - POST with missing X-ACR-Secret header -> 200 response (silent drop, NOT enqueued)

    - POST with valid body matching AcrCloudCallbackSchema -> accepted (200)

    - POST with missing data object -> 400 validation error
    - POST with empty body -> 400 validation error

    Enqueue tests:
    - Valid callback body is enqueued with job name "process-callback"
    - Enqueued job data matches the raw callback payload (no transformation)
    - Job options include removeOnComplete: 1000 and removeOnFail: 5000

    Response tests:
    - Valid request returns { status: "ok" }
    - Invalid auth returns { status: "ok" } (same response, no information leak)
